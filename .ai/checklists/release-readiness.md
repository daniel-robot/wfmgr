# Release Readiness Checklist

A change is release-ready when:

- [ ] Requirements are clear.
- [ ] Architecture impact has been reviewed if needed.
- [ ] Implementation is complete.
- [ ] Tests or validation have been completed.
- [ ] Regression risks are addressed or documented.
- [ ] Known issues are documented.
- [ ] Documentation is updated if needed.
- [ ] Deployment or configuration impact is understood.
- [ ] Rollback or recovery notes exist if needed.
