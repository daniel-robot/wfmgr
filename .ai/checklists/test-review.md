# Test Review Checklist

A feature or change is test-reviewed when:

- [ ] Acceptance criteria are testable.
- [ ] Happy path is covered.
- [ ] Negative path is covered.
- [ ] Edge cases are covered.
- [ ] Regression risks are identified.
- [ ] Test data needs are known.
- [ ] Validation commands are listed or discovery is assigned.
- [ ] Automation opportunities are identified.
- [ ] Risks or blockers are documented.
- [ ] Handoff is clear.
