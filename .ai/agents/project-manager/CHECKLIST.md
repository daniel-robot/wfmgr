# Project Manager Checklist

## Sprint Planning

- [ ] Is the sprint goal clear?
- [ ] Are candidate backlog items clear?
- [ ] Are items small enough?
- [ ] Are dependencies known?
- [ ] Is each task assigned to one owner agent?
- [ ] Are risks identified?
- [ ] Is the definition of done clear?
- [ ] Is validation expected for implementation tasks?

## Task Breakdown

- [ ] Is the task small?
- [ ] Is the owner agent clear?
- [ ] Is the expected output clear?
- [ ] Are affected files or areas identified?
- [ ] Are dependencies listed?
- [ ] Are constraints listed?
- [ ] Is the next step actionable?

## Handoff Quality

- [ ] Does the handoff state the current goal?
- [ ] Does the handoff summarize current state?
- [ ] Does the handoff list relevant files or folders?
- [ ] Does the handoff say what not to do?
- [ ] Does the handoff include expected output?
- [ ] Does the handoff include validation commands, if relevant?
- [ ] Is the next agent clear?

## Risk Tracking

- [ ] Are technical risks tracked?
- [ ] Are product risks tracked?
- [ ] Are delivery risks tracked?
- [ ] Are environment risks tracked?
- [ ] Are file edit conflict risks tracked?
- [ ] Are mitigations identified?

## File Conflict Prevention

- [ ] Has .ai/locks been checked?
- [ ] Are file ownership boundaries clear?
- [ ] Is only one implementation agent editing a file area?
- [ ] Is a lock needed?
- [ ] Has handoff been updated after work?
