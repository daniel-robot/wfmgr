# Product Owner Checklist

## Story Readiness

- [ ] Is the user/persona clear?
- [ ] Is the business goal clear?
- [ ] Is the user value clear?
- [ ] Is the story small enough?
- [ ] Is the story independently deliverable?
- [ ] Are assumptions documented?
- [ ] Are dependencies documented?
- [ ] Is out-of-scope work documented?
- [ ] Is the next agent clear?

## Acceptance Criteria

- [ ] Are acceptance criteria testable?
- [ ] Are acceptance criteria observable?
- [ ] Are happy paths covered?
- [ ] Are negative paths covered?
- [ ] Are validation rules covered?
- [ ] Are permission rules covered?
- [ ] Are error states covered?
- [ ] Are edge cases covered?
- [ ] Is expected system behavior clear?

## Backlog Refinement

- [ ] Is the backlog item clear?
- [ ] Is the priority clear?
- [ ] Is the scope controlled?
- [ ] Is the item ready for Architect review?
- [ ] Is the item ready for implementation after design review?
- [ ] Should this item be split?
- [ ] Are blockers identified?

## Business Rule Review

- [ ] Are required fields clear?
- [ ] Are optional fields clear?
- [ ] Are status/state changes clear?
- [ ] Are role-based permissions clear?
- [ ] Are audit/history expectations clear?
- [ ] Are notification expectations clear?
- [ ] Are data retention or compliance expectations clear?

## Edge Case Review

- [ ] Empty input
- [ ] Invalid input
- [ ] Duplicate input
- [ ] Missing data
- [ ] Permission denied
- [ ] Concurrent update
- [ ] External service failure
- [ ] Network failure
- [ ] Partial save or partial completion
- [ ] User cancellation
- [ ] Long-running operation
